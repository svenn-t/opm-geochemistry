## Equilibrium between seawater, CO2 and calcite
Consider the following input file, `seawater_eq.dat`

```
GEOCHEM
Temp 25
Pres 1.01325e5
debug 0

SOLUTION 0
pH 7 charge
Na 0.48
K 0.01
Mg 0.054
Ca 0.01
SO4 0.028
HCO3 0.00267
Cl 0.56
/end


EQUILIBRIUM_PHASES
calcite 1 0
CO2(G) 1 -3.5
/end

chemtol
1e-8  1e-5
/end
/end
```

*Explanation of input file:*

`GEOCHEM .. /end`:
  :    
  Defines a GEOCHEM calculation block.
`Temp`:
  :    
  temperature of solution, units: Celsius, default 25 $^\circ$C
`Pres`:
  :    
  pressure of solution, units: Pascal, default 10$^5$ Pa
`debug`:
  :    
  controls level of debugging
  * 0 No additional debug information
  * 1 Prints debug data to screen and some extra files

`SOLUTION <name>`:
  :    
  see [solution example](../../solution/html/solution-readable.md).
`EQULIBRIUM_PHASES .. /end`:
  :    
  Mineral and gas buffers are specified by listing the name of the phase followed by two numerical parameters. The phase name may be given either as the full name or as a recognized abbreviation (nick name). A complete list of available mineral and gas phases is provided in the appendix.
  * The first numerical parameter represents the amount of the mineral or gas phase. The second parameter specifies the target saturation index. For gas phases, this value corresponds to the logarithm of the gas partial pressure.
  * In equilibrium calculations, the numerical value of the amount does not influence the calculated equilibrium, provided that it is greater than zero. The amount serves only to indicate that the phase is present and available to equilibrate with the aqueous solution.



```
Terminal> GeoChemX EQSOLVER seawater_eq.dat
```

This table shows how much calcite has been formed (positive sign) from solution, which indicates that this seawater composition is slightly super saturated with respect to calcite. Three values are listed mmol/kg-rock, $\text{wt}\%$, and mol/kgw. The first and second should be equal (except for the conversion factor), both are listed as a sanity check of the code as internally they are calculated differently. Note that it is implicitly assumed that there is an equal amount of rock and water in the system (1 kg). The $\text{wt}\%$ is simply taking mmol/kg-rock, and multiply with molar weight i.g. for $\mathrm{CO_2}$: $1.28086\cdot 1	0^{-3}$ mol $\times$ 44.01 g/mol = 0.0563 g, assuming 1 kg of rock we find a weight percent of 0.00563.

At the end of the file saturation indexes are listed for *all minerals that can be formed from the basis species in the solution*. They are listed from high to low, all minerals with a saturation index larger than zero can theoretically be formed. The table should be checked to see that the solution indeed matches the input, i.e. that calcite has a saturation index of zero and $\mathrm{CO_2}$ and index of -3.5.
